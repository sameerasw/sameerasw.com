### 4.2.1 IP address not found (N/A)

- If the mac app shows it’s IP address as N/A, that means it could not find a local IP address to start to websocket server with.
- NetSticks, USB dongles or anything that gives a direct internet connection rather than connecting through a local area network can cause this.
- Use the drop down menu for networks in app settings to pick available networks if there any others present.
- If more issues are present or your default network adapter not being automatically being detected, please reach out to the developer or create a new issue on GitHub. Your feedback is always helpful. <3

### 4.2.2 Connected only from Android’s side.

- Please check if your apps are up to date as now we are using an end-to-end encrypted connection. If the Android app does not support it, Will show similar issues.

### 4.2.3 Random disconnects

- There are no known issues for this behavior. If you are experiencing, Please check if your power saving or data saving policies causes network throttling as example when the screen is off and such. If no clues were present, please reach out for support in the r/AirSync sub-reddit or in GitHub.
